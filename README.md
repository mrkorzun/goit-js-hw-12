## Про проєкт

Застосунок є розширеною версією проєкту з
[модуля 11](https://github.com/mrkorzun/goit-js-hw-11). Реалізовано підтримку
пагінації через кнопку **Load more**: користувач може послідовно підвантажувати
наступні групи зображень, не перезавантажуючи сторінку. Код повністю
відрефакторено з `.then()/.catch()` на синтаксис **async/await**.

Ключові акценти проєкту:

- рефакторинг асинхронного коду на синтаксис **async/await**;
- пагінація: параметри `page` і `per_page` у запитах до
  [Pixabay API](https://pixabay.com/service/about/api/);
- кнопка **Load more** для підвантаження наступної сторінки результатів;
- визначення кінця колекції через `totalHits` та відповідне повідомлення;
- плавне прокручування сторінки після кожного підвантаження через
  `window.scrollBy`;
- галерея з модальним переглядом на базі **SimpleLightbox**.

## Опис модулів

### `pixabay-api.js`

Модуль, що відповідає за взаємодію з бекендом. <br> Експортує функцію виконання
HTTP-запиту до Pixabay API з підтримкою пагінації.

| Функція                         | Параметри                       | Повертає                            |
| ------------------------------- | ------------------------------- | ----------------------------------- |
| `getImagesByQuery(query, page)` | `query: string`, `page: number` | Значення `data` з відповіді сервера |

### `render-functions.js`

Модуль, що відповідає за рендеринг інтерфейсу та керування галереєю. <br>
Містить екземпляр `SimpleLightbox` та функції роботи з DOM.

| Функція                 | Параметри       | Опис                                                                                  |
| ----------------------- | --------------- | ------------------------------------------------------------------------------------- |
| `createGallery(images)` | `images: Array` | Створює розмітку карток і додає її в контейнер галереї, викликає `lightbox.refresh()` |
| `clearGallery()`        | —               | Очищає вміст контейнера галереї перед новим пошуком                                   |
| `showLoader()`          | —               | Додає клас, що відображає індикатор завантаження                                      |
| `hideLoader()`          | —               | Видаляє клас, що приховує індикатор завантаження                                      |
| `showLoadMoreButton()`  | —               | Додає клас, що відображає кнопку Load more                                            |
| `hideLoadMoreButton()`  | —               | Видаляє клас, що приховує кнопку Load more                                            |

### `main.js`

Точка входу застосунку. <br>Імпортує функції з модулів `pixabay-api.js` та
`render-functions.js`, описує всю бізнес-логіку:

- обробку події `submit` на формі пошуку та `click` на кнопці Load more;
- збереження пошукового запиту в глобальній змінній;
- лічильник поточної сторінки `page` з початковим значенням `1`;
- скидання `page` до `1` при новому сабміті форми;
- послідовність викликів при сабміті: `clearGallery()` → `hideLoadMoreButton()`
  → `showLoader()` → запит → `hideLoader()` → `createGallery()` →
  `showLoadMoreButton()`;
- визначення кінця колекції через `totalHits` та виведення
  `iziToast`-повідомлення;
- плавне прокручування сторінки через `window.scrollBy` після кожного
  підвантаження;
- обробку порожньої відповіді та помилок HTTP-запитів через `iziToast`.

## Функціональність

- При сабміті форми виконується HTTP-запит до Pixabay API. Галерея очищається,
  кнопка Load more ховається, відображається індикатор завантаження.
- Після отримання відповіді індикатор зникає, у DOM додається розмітка карток
  **за одну операцію**, з'являється кнопка Load more.
- За кліком на кнопку **Load more** виконується запит на наступну сторінку
  (`page` збільшується на `1`), нові картки **додаються** до існуючих.
- Після кожного підвантаження сторінка плавно прокручується вниз на дві висоти
  картки галереї через `window.scrollBy`.
- Якщо користувач досяг кінця колекції — кнопка ховається та виводиться
  повідомлення:

  ```
  We're sorry, but you've reached the end of search results.
  ```

- Якщо результатів немає — виводиться `iziToast`-повідомлення:

  ```
  Sorry, there are no images matching your search query. Please try again!
  ```

- При кліку на картку відкривається повнорозмірне зображення в модальному вікні
  `SimpleLightbox`.
- Усі помилки HTTP-запитів обробляються в блоці `catch` та виводяться
  користувачу через `iziToast`.

## Параметри HTTP-запиту

Запит до Pixabay API формується з обов'язковим набором параметрів:

| Параметр      | Значення           | Призначення                        |
| ------------- | ------------------ | ---------------------------------- |
| `key`         | особистий API-ключ | Унікальний ключ доступу до сервісу |
| `q`           | рядок із форми     | Пошуковий запит користувача        |
| `image_type`  | `photo`            | Лише фотографії                    |
| `orientation` | `horizontal`       | Горизонтальна орієнтація           |
| `safesearch`  | `true`             | Фільтр безпечного контенту         |
| `per_page`    | `15`               | Кількість зображень на сторінку    |
| `page`        | поточна сторінка   | Номер сторінки для пагінації       |

## Використані властивості об'єкта зображення

Зі структури об'єкта, що повертає Pixabay, в галереї використовуються:

| Властивість     | Призначення                            |
| --------------- | -------------------------------------- |
| `webformatURL`  | Маленьке зображення для картки галереї |
| `largeImageURL` | Велике зображення для модального вікна |
| `tags`          | Опис — підставляється в атрибут `alt`  |
| `likes`         | Кількість вподобайок                   |
| `views`         | Кількість переглядів                   |
| `comments`      | Кількість коментарів                   |
| `downloads`     | Кількість завантажень                  |
| `totalHits`     | Загальна кількість результатів пошуку  |

## Встановлення та запуск

### Клонування репозиторію

```bash
git clone https://github.com/mrkorzun/goit-js-hw-12
cd goit-js-hw-12
```

### Встановлення залежностей

```bash
npm install
```

### Запуск dev-сервера

```bash
npm run dev
```

### Збірка продакшн-версії

```bash
npm run build
```

### Деплой на GitHub Pages

```bash
npm run deploy
```
